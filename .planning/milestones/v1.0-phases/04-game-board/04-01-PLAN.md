3. Create `src/stores/gameStore.ts` -- Zustand store with immer middleware:

   State interface `GameState`:
   - `players: Record<number, PlayerDto>` (plain object, NOT Map -- avoids immer Map serialization pitfall)
   - `cards: Record<number, CardDto>` (plain object for O(1) lookup)
   - `stack: SpellAbilityDto[]`
   - `combat: CombatDto | null`
   - `phase: string | null`
   - `turn: number`
   - `activePlayerId: number`
   - `humanPlayerId: number | null`
   - `prompt: PromptState | null` (where `PromptState = { type: OutboundMessageType; inputId: string; payload: PromptPayload }`)
   - `buttons: ButtonPayload | null`
   - `gameOver: { winner: string; message: string } | null`
   - `connected: boolean`
   - `error: string | null`

   Actions:
   - `applyGameState(dto: GameStateDto)`: Clear and replace all state. Set `humanPlayerId` from first BUTTON_UPDATE's `playerId` if not yet set. Convert `dto.players` array to Record keyed by id. Convert `dto.cards` array to Record keyed by id.
   - `applyZoneUpdate(updates: ZoneUpdateDto[])`: Mark zones as updated. Per RESEARCH.md pitfall 1, the actual card data comes from card updates, not zone notifications. Update player zone arrays from the cards' `zoneType` field by re-deriving zone membership.
   - `applyPhaseUpdate(phase: string)`: Set `state.phase = phase`.
   - `applyTurnUpdate(payload: { turn: number; activePlayerId: number })`: Set turn and activePlayerId.
   - `applyCombatUpdate(combat: CombatDto | null)`: Set `state.combat = combat`.
   - `applyStackUpdate(stack: SpellAbilityDto[])`: Set `state.stack = stack`.
   - `applyButtonUpdate(buttons: ButtonPayload)`: Set `state.buttons = buttons`. If `humanPlayerId` is null, set it to `buttons.playerId`.
   - `setPrompt(prompt: PromptState | null)`: Set `state.prompt = prompt`.
   - `setGameOver(payload: { winner: string; message: string })`: Set `state.gameOver = payload`.
   - `setConnected(connected: boolean)`: Set `state.connected = connected`.
   - `setError(error: string | null)`: Set `state.error = error`.
   - `reset()`: Reset all state to initial values.

   Export `useGameStore` created via `create<GameState & GameActions>()(immer((set) => ({...})))`.

     private ws: WebSocket | null = null

     private maxReconnectAttempts = 3

     constructor(gameId: string) { this.gameId = gameId }

     connect(): void {
       // Connect to ws://${location.host}/ws/game/${this.gameId}

       // On close: store.setConnected(false), attempt reconnect
       // On error: store.setError(message)
       // On message: parse JSON as OutboundMessage, switch on msg.type:
       //   GAME_STATE -> store.applyGameState(msg.payload as GameStateDto)
       //   ZONE_UPDATE -> store.applyZoneUpdate(msg.payload as ZoneUpdateDto[])
       //   PHASE_UPDATE -> store.applyPhaseUpdate((msg.payload as {phase: string}).phase)
       //   TURN_UPDATE -> store.applyTurnUpdate(msg.payload as {turn: number, activePlayerId: number})
       //   COMBAT_UPDATE -> store.applyCombatUpdate(msg.payload as CombatDto | null)
       //   STACK_UPDATE -> store.applyStackUpdate(msg.payload as SpellAbilityDto[])
       //   BUTTON_UPDATE -> store.applyButtonUpdate(msg.payload as ButtonPayload)
       //   PROMPT_CHOICE, PROMPT_CONFIRM, PROMPT_AMOUNT -> store.setPrompt({type: msg.type, inputId: msg.inputId!, payload: msg.payload as PromptPayload})
       //   GAME_OVER -> store.setGameOver(msg.payload as {winner: string, message: string})
       //   ERROR -> store.setError((msg.payload as {message: string}).message)
       //   MESSAGE, SHOW_CARDS -> log but no store update for v1
     }

     }
