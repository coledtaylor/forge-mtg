# Phase 5 — UI Design Contract

> Visual and interaction contract for the game setup lobby and deck-to-game integration flow. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase scope reminder:** This phase delivers the lobby screen (format selection, deck picker, AI settings, start game), the "Play This Deck" shortcut from deck editor, and the "Return to Lobby" flow from game over. No changes to the game board or deck builder visuals themselves.

## Layout Contract

### Lobby Screen Structure

Centered single-column layout. The lobby panel sits in the center of the viewport, vertically offset 48px from top. Maximum width 480px. This is a focused, clean pre-game screen -- not a dashboard.

```
+------------------------------------------------------------------+
|                                                                    |
|                    +--------------------------+                    |
|                    |    Play a Game           |                    |
|                    |                          |                    |
|                    |  Format: [Commander  v]  |                    |
|                    |                          |                    |
|                    |  Your Decks              |                    |
|                    |  ( ) Mono Red Aggro  60  |                    |
|                    |  (*) Elves Combo     60  |                    |
|                    |  ( ) Control Shell   60  |                    |
|                    |                          |                    |
|                    |  > AI Settings            |                    |
|                    |    Default · Random deck  |                    |
|                    |                          |                    |
|                    |  [    Start Game      ]  |                    |
|                    +--------------------------+                    |
|                                                                    |
+------------------------------------------------------------------+
```

### Lobby Panel Dimensions

| Property | Value |
|----------|-------|
| Max width | 480px |
| Padding | 32px (xl token) |
| Top offset | 48px from viewport top (2xl token) |
| Horizontal centering | `mx-auto` |
| Background | `--card` surface with `rounded-lg` border radius |
| Border | 1px `--border` |

### Deck Picker List

| Property | Value |
|----------|-------|
| Max visible items | 6 decks before scrolling |
| Item height | 44px (comfortable click target) |
| Item padding | 8px horizontal (sm), 0 vertical (height handles vertical) |
| Selected item | Accent left border (3px) + subtle accent background (`bg-primary/10`) |
| Unselected item | Transparent background, muted border-bottom (1px) |
| Scroll | `overflow-y-auto`, `max-h-[264px]` (6 items x 44px) |

### AI Settings Collapsible

| Property | Value |
|----------|-------|
| Default state | Collapsed |
| Collapsed view | Clickable row: chevron icon + "AI Settings" label + summary text ("Default difficulty . Random deck") |
| Expanded view | Two dropdowns: Difficulty (Easy/Medium/Hard), AI Deck (Random/specific deck names) |
| Animation | Height transition 200ms ease-out |
| Internal spacing | 16px (md) between the two dropdowns |

## Interaction States

### Format Selector

| State | Visual |
|-------|--------|
| Default | shadcn Select with placeholder "Select a format" |
| Open | Dropdown with 4 options: Commander, Standard, Casual 60-card, Jumpstart |
| Selected | Shows selected format name, triggers deck list filter |
| Pre-filled (from deck editor) | Format auto-selected from deck's format field |

### Deck Picker

| State | Visual |
|-------|--------|
| Loading | Skeleton lines (3 rows) inside the picker area |
| Empty (no decks for format) | Message: "No decks for {format}." + link text "Create one in the Deck Builder." |
| Populated | List of deck names with card count badge, radio-style selection |
| Unselected item | Transparent background, muted bottom border |
| Selected item | Accent left border (3px), subtle accent background (`bg-primary/10`) |
| Hover (unselected) | Background changes to `bg-muted` |
| Pre-selected (from deck editor) | Deck auto-selected, scroll position adjusted to show it |

### Start Game Button

| State | Visual |
|-------|--------|
| Disabled (no deck selected) | Primary variant with `opacity-50`, `cursor-not-allowed` |
| Enabled (deck selected) | Primary variant, full opacity, accent background |
| Loading (game starting) | Button shows spinner + "Starting..." text, disabled to prevent double-click |

### AI Settings Toggle

| State | Visual |
|-------|--------|
| Collapsed | Single row: chevron-right icon + "AI Settings" in label size + summary in muted-foreground |
| Expanded | Chevron rotates to chevron-down, two Select dropdowns appear below with 200ms height transition |
| Summary text (collapsed) | Shows current selections: "{difficulty} difficulty . {deck}" in muted-foreground |

### Play This Deck Button (Deck Editor)

| State | Visual |
|-------|--------|
| Default | Primary variant button in deck editor header, "Play This Deck" label |
| No format on deck | Button still works, navigates to lobby with format unset (user picks format in lobby) |

## Navigation Flow Contract

### Happy Path (Lobby Direct)

1. User clicks "Play a Game" from deck list header (or navigation)
2. Lobby opens with no format pre-selected, "Start Game" disabled
3. User selects format from dropdown
4. Deck list filters to matching decks
5. User clicks a deck (radio selection, one at a time)
6. "Start Game" button becomes enabled
7. User clicks "Start Game" -- button shows "Starting..." spinner
8. Game board loads with selected deck

### Happy Path (From Deck Editor)

1. User clicks "Play This Deck" in deck editor header
2. Deck auto-saves (flushSave)
3. Lobby opens with format pre-filled and deck pre-selected
4. User clicks "Start Game" (one click to play)

### Return to Lobby After Game

1. Game ends, GameOverScreen shows "Return to Lobby" button
2. User clicks "Return to Lobby"
3. Lobby opens with the same format and deck pre-selected (one click to rematch)
