Create two handler classes and register them in WebServer. Then add integration tests.

**CardSearchHandler.java** (`forge.web.api` package):
- Single static method: `public static void search(Context ctx)`
- Extract query params: `q` (String), `color` (String), `type` (String), `cmc` (String, parsed to int), `cmcOp` (String, default "eq"), `format` (String), `page` (int, default 1), `limit` (int, default 20)
- Build `Predicate<PaperCard>` by composing:
  - If `q` is non-null/non-empty: `PaperCardPredicates.searchableName(PredicateString.StringOp.CONTAINS_IC, q)`
  - If `color` is non-null: For each character in color string (e.g., "WR"), build a predicate that checks `pc.getRules().getColorIdentity()` has that color. Use AND logic (card must contain ALL specified colors). Access color via `ColorSet` methods: 'W' -> `hasWhite()`, 'U' -> `hasBlue()`, 'B' -> `hasBlack()`, 'R' -> `hasRed()`, 'G' -> `hasGreen()`
  - If `type` is non-null: `PaperCardPredicates.fromRules(CardRulesPredicates.coreType(true, CardType.CoreType.valueOf(type)))` -- type param should be "Creature", "Instant", "Sorcery", etc.
  - If `cmc` is non-null: Parse cmc to int, map cmcOp string to `ComparableOp`: "eq" -> EQUALS, "lt" -> LESS_THAN, "gt" -> GREATER_THAN, "lte" -> LESS_OR_EQUAL, "gte" -> GREATER_OR_EQUAL. Then: `PaperCardPredicates.fromRules(CardRulesPredicates.cmc(op, cmcValue))`
  - If `format` is non-null: `GameFormat fmt = FModel.getFormats().get(format); if (fmt != null) filter = filter.and(fmt.getFilterRules());`
- Apply filter to `FModel.getMagicDb().getCommonCards().getUniqueCards().stream()`
- Sort by `PaperCard::getName` (case-insensitive: `String.CASE_INSENSITIVE_ORDER`)
- Collect to list, compute `totalCount`, then `.skip((page-1)*limit).limit(limit).map(CardSearchDto::from)`
- Return JSON: `Map.of("cards", pageResults, "total", totalCount, "page", page, "limit", limit, "totalPages", (int) Math.ceil((double) totalCount / limit))`

**DeckHandler.java** (`forge.web.api` package):
- Five static methods: `list`, `create`, `get`, `update`, `delete` -- all take `Context ctx`
- `private static File getDecksDir()`: returns `new File(ForgeConstants.DECK_CONSTRUCTED_DIR)`
- `list(Context ctx)`:
  - Recursively scan getDecksDir() for `.dck` files
  - For each file: `DeckSerializer.fromFile(file)` -> `DeckSummaryDto.from(deck, relativePath)` where relativePath is path relative to decks dir
  - Return JSON array of DeckSummaryDto
- `create(Context ctx)`:
  - Parse request body as a JSON object with `name` (String) field. Use `ctx.bodyAsClass(Map.class)` or a simple inner request class.
  - Create: `Deck deck = new Deck(name)`
  - Write: `DeckSerializer.writeDeck(deck, new File(getDecksDir(), deck.getBestFileName() + ".dck"))`
  - Ensure deck directory exists: `getDecksDir().mkdirs()`
  - Return 201 with DeckDetailDto.from(deck)
- `get(Context ctx)`:
  - `String name = ctx.pathParam("name")` (URL-decoded by Javalin)
  - Search for matching .dck file by name in getDecksDir()
  - If not found: `ctx.status(404).json(Map.of("error", "Deck not found"))`
  - Else: `ctx.json(DeckDetailDto.from(DeckSerializer.fromFile(deckFile)))`
- `update(Context ctx)`:
  - Get existing deck file by name
  - Parse body for updated card lists (main, sideboard sections as `Map<String, Integer>` cardName -> quantity)
  - Load deck, clear sections, re-add cards from request body using `FModel.getMagicDb().getCommonCards().getCard(cardName)`
  - Write updated deck back to same file
  - Return 200 with DeckDetailDto
- `delete(Context ctx)`:
  - Find deck file by name, delete it
  - Return 204 (no content) on success, 404 if not found
