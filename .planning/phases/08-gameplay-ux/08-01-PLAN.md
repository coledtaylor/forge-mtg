**3a. GameLog streaming via delta pattern:**
Add a `private int lastLogIndex = 0;` field.

Add a `sendLogDelta()` method:
```java
private void sendLogDelta() {
    final GameView gv = getGameView();
    if (gv == null) return;
    final forge.game.GameLog gameLog = gv.getGameLog();
    if (gameLog == null) return;

    final java.util.List<forge.game.GameLogEntry> allEntries = gameLog.getLogEntries(null);
    final int totalEntries = allEntries.size();
    if (totalEntries <= lastLogIndex) return;

    // allEntries is in REVERSE order (newest first), so new entries are at indices 0..(newCount-1)
    final int newCount = totalEntries - lastLogIndex;
    final java.util.List<Map<String, Object>> entries = new ArrayList<>();
    for (int i = newCount - 1; i >= 0; i--) {
        final forge.game.GameLogEntry entry = allEntries.get(i);
        entries.add(payloadMap(
            "type", entry.type().name(),
            "message", entry.message(),
            "sourceCardId", entry.sourceCard() != null ? entry.sourceCard().getId() : -1
        ));
    }
    lastLogIndex = totalEntries;
    send(MessageType.GAME_LOG, entries);
}
```

Call `sendLogDelta()` at the end of these existing fire-and-forget methods: `updateZones`, `updatePhase`, `updateStack`, `updateButtons` (the one that sends BUTTON_UPDATE), `showCombat`. Add the call as the last line in each method body. Check if `entry.sourceCard()` returns a GameEntityView -- if `GameLogEntry` does not have `sourceCard()`, use -1 for all entries and drop that field.

NOTE: Read `GameLogEntry.java` and `GameLogEntryType.java` first to understand the exact API. `GameLog.getLogEntries(null)` returns entries filtered by type (null = all). Confirm the method signature and return type.
