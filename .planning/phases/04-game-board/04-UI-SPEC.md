# Phase 4 — UI Design Contract

> Visual and interaction contract for the game board. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase scope reminder:** This phase delivers the full gameplay UI -- battlefield, zones, prompts, combat, and real-time WebSocket updates. The deck builder (Phase 3) and game setup lobby (Phase 5) are separate phases. This contract covers only the in-game board surfaces.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (already initialized) |
| Preset | base-nova, dark mode, neutral base color |
| Component library | base-ui (via shadcn/ui) |
| Icon library | lucide-react |
| Font | Inter (system font stack fallback: `ui-sans-serif, system-ui, sans-serif`) |

**Existing assets reused from Phases 2-3:**
- `CardImage` component (Scryfall images with lazy loading)
- `CardHoverPreview` + `useCardHover` (floating preview near cursor)
- `ManaCost` component + mana-font CSS (mana symbol rendering)
- shadcn `button`, `badge`, `dialog`, `tooltip`, `skeleton` components
- All CSS variables in `index.css` (dark theme established)

**Note on card images in game context:** `CardDto` from the engine lacks `setCode`/`collectorNumber`. The game board must use Scryfall's name-based image API (`https://api.scryfall.com/cards/named?exact={name}&format=image&version=normal`) or a name-to-image cache. The existing `CardImage` component requires adaptation or a new `GameCardImage` wrapper that accepts a card name instead of set/collector data.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, counter badge padding, mana symbol gaps in pool display |
| sm | 8px | Card-to-card gap in hand fan, attachment offset, zone internal padding |
| md | 16px | Zone section padding, action bar internal padding, stack panel gap |
| lg | 24px | Layout gaps between major zones (battlefield to phase strip) |
| xl | 32px | Not used in Phase 4 (board is dense, not spacious) |
| 2xl | 48px | Not used in Phase 4 |
| 3xl | 64px | Not used in Phase 4 |

Exceptions:
- Hand cards overlap by 40px (negative margin) in the fan layout, with 60px visible per card
- Touch/click target for hand cards: 60px wide minimum (the visible portion)
- Battlefield card size: 100px wide x 140px tall (smaller than deck builder's 244px to fit many permanents)
- Tapped card rotation uses `transform: rotate(90deg)` with the card's center as origin
- Attacker forward shift: 12px toward center line
- Blocker shift: 8px toward assigned attacker

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |

**Font weights used:** 400 (regular) and 600 (semibold). No other weights permitted.

**Phase 4 specific usage:**
- **Body (14px/400):** Prompt text in action bar, choice option labels, stack spell descriptions
- **Label (12px/400):** Player name, life total, hand count, mana quantities, counter badge text, phase strip labels, P/T overlay, zone count badges
- **Heading (20px/600):** Life total number (large for readability), game over title

Display size (28px) is not used in Phase 4. The game board is information-dense; all text stays at Body or Label size except life totals and game-over.

## Color

Inherits all CSS variables from the established dark theme in `index.css`. No new CSS variables added.

| Role | CSS Variable | oklch Value | Usage |
|------|-------------|-------------|-------|
| Dominant (60%) | `--background` | `oklch(0.15 0.005 285)` | Board background, zone backgrounds |
| Secondary (30%) | `--card` | `oklch(0.25 0.003 285)` | Player info bars, action bar, stack panel, zone overlays |
| Accent (10%) | `--primary` | `oklch(0.637 0.194 255)` | See reserved list below |
| Destructive | `--destructive` | `oklch(0.45 0.15 25)` | Damage flash, blocker removal |

**Accent reserved for:**
- Phase strip: current phase highlight background
- Active prompt: action bar border-top glow when awaiting player input
- Targeting mode: valid target card glow/border (2px accent outline)
- OK/Confirm button in action bar (primary variant)
- Turn indicator for "your turn"

**NOT accent:** Card borders at rest, zone labels, mana symbols, life total text, player name text.

**Primary focal point:** During gameplay, the action bar is the primary focal point when a prompt is active; the active player's battlefield is the primary focal point otherwise.

### Game-Specific Semantic Colors

These are applied via Tailwind utility classes, not new CSS variables:

| Purpose | Color | Tailwind Class | Usage |
|---------|-------|----------------|-------|
| Attacker glow | Red-orange | `shadow-red-500/50` + `ring-2 ring-red-500` | Cards declared as attackers |
| Blocker assignment line | Yellow | `stroke-yellow-400` (SVG) | Arrow/line from blocker to attacker |
| Damage flash | Red | `bg-red-500/30` | Brief overlay on creatures taking damage |
| Life loss animation | Red text | `text-red-400` | Life total counting down |
| Life gain animation | Green text | `text-green-400` | Life total counting up |
| Targeting valid | Accent glow | `ring-2 ring-primary` | Cards that are valid targets |
| Targeting invalid | Dimmed | `opacity-40` | Cards that cannot be targeted |
| Creature dying | Fade out | `opacity-0 transition-opacity duration-500` | Creature leaving battlefield |
| Card playable | Subtle glow | `ring-1 ring-primary/40` | Cards in hand that can currently be played |

## Layout Contract

### Board Structure

Arena-style layout. Opponent at top, player at bottom, battlefield center. Full viewport (100vh x 100vw), no scrolling on the main board.

```
+------------------------------------------------------------------+
|  [Opponent Info Bar] Name | Life: 20 | Hand: 7 | {mana symbols}  |
+------------------------------------------------------------------+
|                                                    |  [Stack]     |
|  [Opponent Battlefield]                            |  Spell 1     |
|    Creatures/Permanents row                        |  Spell 2     |
|    Lands row                                       |  ...         |
|                                                    |              |
+--[GY]--[Exile]-----[Phase Strip]-------------[Lib]+              |
|                                                    |              |
|  [Player Battlefield]                              |              |
|    Creatures/Permanents row                        |              |
|    Lands row                                       |              |
|                                                    |              |
+--[GY]--[Exile]-------------------------------------+-------------+
|  [Action Bar] "Choose a target" [OK] [Cancel] [Pass Priority]    |
+------------------------------------------------------------------+
|  [Hand Zone]  card card card card card card card                  |
+------------------------------------------------------------------+
|  [Player Info Bar] Name | Life: 20 | {mana symbols}              |
+------------------------------------------------------------------+
```

### CSS Grid Structure

```
grid-template-rows: 36px 1fr 32px 1fr 44px auto 36px
grid-template-columns: 1fr 220px
```

| Row | Height | Content |
|-----|--------|---------|
| 1 | 36px | Opponent info bar (spans both columns) |
| 2 | 1fr | Opponent battlefield (col 1), Stack panel (col 2, spans rows 2-4) |
| 3 | 32px | Phase strip + center divider (spans both columns) |
| 4 | 1fr | Player battlefield (col 1) |
| 5 | 44px | Action bar (spans both columns) |
| 6 | auto | Hand zone -- height based on card count, max 160px (spans both columns) |
| 7 | 36px | Player info bar (spans both columns) |

### Zone Placement

- **Graveyard piles:** Left side of center divider row, one per player (opponent GY above divider, player GY below)
- **Exile piles:** Next to graveyard piles
- **Library piles:** Right side of center divider row (one per player)
- **Stack panel:** Fixed 220px width right column, spanning opponent battlefield through player battlefield rows

### Minimum Viewport

1280px wide (per REQUIREMENTS.md -- desktop only, no mobile responsive). Board scales to fill viewport.

## Interaction States

### Hand Cards

| State | Visual |
|-------|--------|
| Default | Card image at 100px wide, overlapping neighbors by 40px, slight arc rotation (max 5deg at edges) |
| Hover | Card raises 40px above hand, scale to 1.1x, z-index above siblings, no arc rotation, hover preview appears |
| Playable | Subtle accent ring (`ring-1 ring-primary/40`) on cards the player can currently cast/play |
| Double-click | Card fades from hand as engine processes the action |
| Not your turn | No playable highlights, hover still works for preview |

### Battlefield Cards

| State | Visual |
|-------|--------|
| Default (untapped) | 100x140px card image, upright |
| Tapped | 90deg clockwise rotation, `transform: rotate(90deg)` |
| Has counters | Colored badge overlay at bottom-right: `+1/+1 x3` in 12px semibold text on dark bg pill |
| Has attachments | Parent card visible, attachments stacked behind offset 8px up + 8px right each, all hoverable |
| Attacker (declared) | Shifts 12px toward center, red glow (`shadow-lg shadow-red-500/50 ring-2 ring-red-500`) |
| Blocker (assigned) | Shifts 8px toward attacker, yellow ring (`ring-2 ring-yellow-400`) |
| Taking damage | Brief red flash overlay (200ms), P/T text updates |
| Dying | Fade out over 500ms (`opacity-0 transition-opacity duration-500`), then removed from DOM |
| Targeting (valid) | Accent glow (`ring-2 ring-primary`), cursor pointer |
| Targeting (invalid) | Dimmed (`opacity-40`), cursor not-allowed |

### Action Bar

| State | Visual |
|-------|--------|
| No prompt | Bar visible but empty, muted background |
| Awaiting input | Accent border-top (2px `border-primary`), prompt text visible, buttons enabled |
| Button focused | Standard shadcn focus ring |
| Auto-passed | Bar shows "Waiting for opponent..." in muted text |

### Player Info Bar

| State | Visual |
|-------|--------|
| Default | Thin 36px bar, card background, label-size text |
| Life changing (loss) | Life number text turns `text-red-400`, counts down over 300ms, returns to foreground color |
| Life changing (gain) | Life number text turns `text-green-400`, counts up over 300ms, returns to foreground color |
| Active turn | Player name has accent color underline or left-border indicator |
| Poison counters > 0 | Poison counter shown as purple badge next to life total |

### Phase Strip

| State | Visual |
|-------|--------|
| Default phase | Label-size text, muted-foreground color |
| Current phase | Accent background pill, primary-foreground text |
| Your turn | Turn number indicator + "Your Turn" text in accent color |
| Opponent turn | Turn number indicator + "Opponent's Turn" in muted-foreground |

### Stack Panel

| State | Visual |
|-------|--------|
| Empty | Panel shows "Stack empty" in muted text, or collapses to just header |
| Has items | Cards listed top-to-bottom (top = next to resolve), each with small card thumbnail (60px wide) + name + description |
| Hover on item | Enlarged card preview appears (reuses hover preview pattern) |

### Zone Piles (Graveyard, Exile, Library)

| State | Visual |
|-------|--------|
| Empty | Outlined placeholder rectangle with zone icon (lucide), 0 badge |
| Has cards | Top card thumbnail (60x84px) with count badge overlay at top-right |
| Hover | Slight scale up (1.05x), tooltip shows zone name + count |
| Click (GY/Exile) | ZoneOverlay dialog opens with scrollable card list |
| Library | No click expansion (hidden information), shows only count |

## Combat UI Contract

### Declare Attackers Phase

1. Eligible creatures (untapped, no summoning sickness) get accent highlight (`ring-2 ring-primary`)
2. Ineligible creatures are dimmed (`opacity-40`)
3. Click an eligible creature to toggle as attacker: shifts 12px forward, red glow applied
4. Click again to un-declare
5. Action bar shows: "Declare attackers" [OK] [Cancel]
6. OK sends confirmation, Cancel sends cancel

### Declare Blockers Phase

1. Opponent's attacking creatures are visible shifted forward with red glow
2. Your eligible blockers get accent highlight
3. Click a blocker, then click an attacker to assign the block
4. SVG arrow drawn from blocker to assigned attacker (yellow `stroke-yellow-400`, 2px stroke width)
5. Click an assigned blocker again to unassign
6. Action bar shows: "Declare blockers" [OK] [Cancel]

### Damage Resolution

1. Creatures taking damage flash red overlay (200ms)
2. P/T overlays update in real-time from WebSocket
3. Creatures that die fade out over 500ms
4. Life totals animate (count up/down over 300ms)

## Animation Timing (Claude's Discretion)

| Animation | Duration | Easing | Trigger |
|-----------|----------|--------|---------|
| Hand card hover raise | 150ms | ease-out | Mouse enter on hand card |
| Hand card lower | 150ms | ease-in | Mouse leave on hand card |
| Card tap/untap rotation | 200ms | ease-in-out | WebSocket tapped state change |
| Attacker shift forward | 200ms | ease-out | Click to declare attacker |
| Blocker shift | 200ms | ease-out | Click to assign blocker |
| Damage flash | 200ms | linear | Damage event from WebSocket |
| Creature death fade | 500ms | ease-out | Card removed from battlefield |
| Life total change | 300ms | ease-out | Life value change from WebSocket |
| Zone overlay open | 200ms | ease-out | Click on zone pile |
| Phase strip highlight | 150ms | ease-in-out | Phase change from WebSocket |

All animations use CSS transitions via Tailwind classes (`transition-transform duration-200 ease-out`, etc.). No JavaScript animation library.

## Card Sizing (Claude's Discretion)

| Zone | Card Width | Card Height | Rationale |
|------|-----------|-------------|-----------|
| Hand | 100px | 140px | Fits 7+ cards with overlap in the hand zone |
| Battlefield | 100px | 140px | Consistent with hand, fits 10+ permanents per row |
| Stack | 60px | 84px | Thumbnail size, text is primary info in stack |
| Zone pile (top card) | 60px | 84px | Small pile indicator, count badge is primary |
| Zone overlay (expanded) | 100px | 140px | Same as battlefield for consistency |
| Hover preview | 260px | auto | Same as Phase 3 CardHoverPreview for consistency |

**Large battlefield handling (Claude's Discretion):** When a player has more than 10 permanents in a single lane (lands or creatures), cards shrink proportionally to fit. Minimum card width: 60px. Below 60px, the lane becomes horizontally scrollable with overflow-x-auto. This prevents the layout from breaking with token-heavy board states.

## WebSocket Error Handling (Claude's Discretion)

| Scenario | User Sees | Behavior |
|----------|-----------|----------|
| Connection lost | Banner: "Connection lost. Reconnecting..." with spinner | Auto-reconnect with exponential backoff: 1s, 2s, 4s (max 3 attempts) |
| Reconnect successful | Banner disappears, board refreshes from GAME_STATE snapshot | Request full GAME_STATE on reconnect to resync |
| Reconnect failed (3 attempts) | Banner: "Could not reconnect to game. Return to lobby." with button | No further auto-retry; user must navigate away |
| Server error message | Banner: "Game error: {message}" | Display for 5 seconds then dismiss, or until next message |

Centered overlay on top of the board (board stays visible but dimmed to `opacity-30`).

| Element | Visual |

| Subtext | 14px regular, muted-foreground, opponent name and game summary |
| Primary action | Button: "Return to Lobby" (primary variant) |
