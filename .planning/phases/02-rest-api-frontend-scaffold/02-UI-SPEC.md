# Phase 2 — UI Design Contract

> Visual and interaction contract for the frontend scaffold. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase scope reminder:** This phase delivers the React scaffold, card search results grid, and Scryfall card image display. The full deck builder UI is Phase 3. The game board is Phase 4. This contract covers only Phase 2 surfaces.

| Property | Value |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between mana symbols |
| sm | 8px | Compact element spacing, card grid gap on dense views |
| md | 16px | Default element spacing, card grid gap, input padding |
| lg | 24px | Section padding, card-to-controls spacing |
| xl | 32px | Layout gaps between major sections (search bar to results) |
| 2xl | 48px | Page-level vertical padding |
| 3xl | 64px | Not used in Phase 2 |

Exceptions: Card images use their natural aspect ratio (488x680, approximately 5:7). Card image display width is 244px (half of Scryfall normal size) with height auto-calculated to maintain ratio.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 500 (medium) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

**Font weights used:** 400 (regular) and 600 (semibold). 500 (medium) is allowed only for labels and small UI text (pagination, filter chips).

**Rationale:** 14px body matches dense data display (card search results with metadata). 12px label handles mana cost, type line, and card count badges without overwhelming card images.

## Color

Built on shadcn dark theme defaults with MTG-inspired accent. These map to shadcn CSS variables in `globals.css`.

| Role | Value | CSS Variable | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `hsl(240 10% 3.9%)` | `--background` | Page background |
| Secondary (30%) | `hsl(240 3.7% 15.9%)` | `--card` | Card containers, search results panel, sidebar |
| Accent (10%) | `hsl(217 91% 60%)` | `--primary` | See reserved list below |
| Destructive | `hsl(0 62.8% 30.6%)` | `--destructive` | Delete deck button, error borders |
| Muted text | `hsl(240 5% 64.9%)` | `--muted-foreground` | Secondary text, type lines, oracle text |
| Border | `hsl(240 3.7% 15.9%)` | `--border` | Card borders, dividers, input borders |

**Accent reserved for:**
- "Search" button / search input focus ring
- Pagination active page indicator
- Card hover border highlight in search results
- Primary action buttons (e.g., "Create Deck")

**NOT accent:** Card text, type line labels, navigation links, filter chip borders.

**Card image fallback colors:**
- Fallback background: `hsl(240 3.7% 12%)` (slightly darker than card surface)
- Fallback border: `hsl(240 3.7% 20%)`
- Fallback card name: foreground color at weight 600
- Fallback mana cost: muted-foreground at weight 400
- Fallback type line: muted-foreground at weight 400

|-----------|------|---------|

| `CardGrid` | `components/CardGrid.tsx` | Responsive grid of CardImage components for search results |

### CardImage Specification

- **Dimensions:** 244px wide, height auto (maintains 488:680 aspect ratio, renders at ~340px)
- **Loading:** `loading="lazy"` on all `<img>` elements
- **Success state:** Scryfall image rendered with 4px border-radius
- **Loading state:** Skeleton placeholder at 244x340px with shimmer animation
- **Error state:** Styled text fallback at 244x340px showing:
  - Card name (14px, semibold, foreground color) — top
  - Mana cost string (12px, regular, muted-foreground) — below name
  - Type line (12px, regular, muted-foreground) — below mana cost
  - Background: fallback card color, border: fallback border color, border-radius: 8px
- **Image source:** `https://api.scryfall.com/cards/{setCode}/{collectorNumber}?format=image&version=normal`

### CardGrid Specification

- **Layout:** CSS Grid, `grid-template-columns: repeat(auto-fill, minmax(244px, 1fr))`
- **Gap:** 16px (md spacing token)
- **Minimum columns:** 3 (at 1280px minimum viewport per requirements)
- **Empty state:** Centered message when no results (see Copywriting Contract)
- **Loading state:** 20 Skeleton placeholders in grid layout (matches default page size)

### SearchBar Specification

- **Layout:** Single row with text input (flex-grow) + filter controls + search button
- **Text input:** Placeholder text "Search cards by name..."
- **Filters:** Select dropdowns for Color, Type, Format. Number input for CMC with operator select (=, <, >, <=, >=)
- **Search trigger:** Button click or Enter key in text input
- **Debounce:** None — search is explicit (button/Enter), not live-as-you-type

### PaginationBar Specification

- **Layout:** Centered row with Previous/Next buttons and page number display
- **Format:** "Page {current} of {total}" with Previous and Next buttons
- **Disabled states:** Previous disabled on page 1, Next disabled on last page
- **Active page:** Accent-colored text for current page number

## Layout Contract

### Page Structure

This phase delivers a single-page layout. Routing is not needed until Phase 3+ when deck builder and game board become separate views.

```
+----------------------------------------------------------+
|  Header: "Forge" logo text (Display size, left-aligned)  |
+----------------------------------------------------------+
|                                                          |
|  SearchBar (full width, xl spacing below header)         |
|  [Search input............] [Color v] [Type v] [Search]  |
|                                                          |
|  Results count: "{N} cards found"        PaginationBar   |
|                                                          |
|  CardGrid (auto-fill responsive grid)                    |
|  +-------+ +-------+ +-------+ +-------+ +-------+      |
|  | Card  | | Card  | | Card  | | Card  | | Card  |      |
|  | Image | | Image | | Image | | Image | | Image |      |
|  |       | |       | |       | |       | |       |      |
|  +-------+ +-------+ +-------+ +-------+ +-------+      |
|  +-------+ +-------+ +-------+ +-------+ +-------+      |
|  | Card  | | Card  | | Card  | | Card  | | Card  |      |

|  |       | |       | |       | |       | |       |      |
```

- **Minimum viewport:** 1280px wide (per REQUIREMENTS.md — desktop only, no mobile responsive)
- **Max content width:** 1200px centered with auto margins
- **Page padding:** 2xl (48px) vertical, lg (24px) horizontal

### Deck List Sidebar (Phase 2 scope: list only, no editing)

The deck list appears as a secondary view. For Phase 2, it is a simple list accessible via navigation. Full deck editing UI is Phase 3.

```
+----------------------------------------------------------+
|  Heading: "Your Decks" (Heading size)     [Create Deck]  |
+----------------------------------------------------------+
|  +------------------------------------------------------+|
|  | Deck Name                            60 cards   WUB  ||
|  +------------------------------------------------------+|
|  | Another Deck                         40 cards   RG   ||
|  +------------------------------------------------------+|
|  | ...                                                   ||
|  +------------------------------------------------------+|
```

- Each deck row shows: name (Body, regular), card count (Label, muted), color identity badges
- Color identity badges: small colored circles (12px) using MTG color hex values
- Hover state: row background shifts to `--accent` at 10% opacity
