1. Create `JumpstartPackDto.java` in `forge-gui-web/src/main/java/forge/web/dto/` with fields:
   - `String id` (full template name, e.g., "JMP Angels 1" -- used by backend for pack loading)
   - `String theme` (display name, e.g., "Angels" -- extracted by stripping set prefix and trailing variant number)
   - `String setCode` ("JMP" or "J22")
   - `int cardCount` (number of cards in the pack)
   - `List<String> colors` (color identity computed from pack cards)
   - Constructor taking all fields, standard getters, no setters (immutable DTO)

2. Create `JumpstartHandler.java` in `forge-gui-web/src/main/java/forge/web/api/` with:
   - `public static void listPacks(final Context ctx)` method
   - Iterate `StaticData.instance().getSpecialBoosters()`
   - Filter templates whose getName() starts with "JMP " or "J22 "
   - For each matching template:
     - Extract setCode = name.substring(0, 3)
     - Extract theme = name.substring(4).replaceAll("\\s+\\d+$", "") to strip trailing variant number
     - Open template via `new UnOpenedProduct(template).get()` to get `List<PaperCard>`
     - Compute color identity: iterate cards, collect unique colors from `card.getRules().getColorIdentity()` using `MagicColor` constants (W/U/B/R/G)
     - Create JumpstartPackDto with (name, theme, setCode, cards.size(), colorsList)
   - Return `ctx.json(packs)` sorted by theme name then set code
   - Import from: `forge.item.SealedTemplate`, `forge.item.generation.UnOpenedProduct`, `forge.item.PaperCard`, `forge.StaticData`, `forge.card.MagicColor`, `forge.card.ColorSet`

3. Register the endpoint in `WebServer.createApp()`:
   - Add `config.routes.get("/api/jumpstart/packs", JumpstartHandler::listPacks);` in the REST endpoints section
   - Add import: `import forge.web.api.JumpstartHandler;`

NOTE on color identity computation: Use `card.getRules().getColorIdentity()` which returns a `ColorSet`. Use `ColorSet.hasWhite()`, `.hasBlue()`, etc., to build the color list as single-letter strings ["W", "U", "B", "R", "G"]. Follow existing color handling patterns in the codebase.

NOTE on determinism: JMP/J22 entries use `wholeSheet` which returns the full print sheet (deterministic). If `UnOpenedProduct.get()` returns different results on repeat calls, log a warning but proceed -- the data is still usable.
