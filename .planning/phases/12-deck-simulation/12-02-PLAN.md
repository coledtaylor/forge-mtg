    1. **SimulationHandler.java** -- Create in forge-gui-web/src/main/java/forge/web/api/:
       - Final utility class with static methods (same pattern as DeckHandler)
       - Uses Jackson ObjectMapper for JSON serialization (same instance pattern as other handlers)

       **POST /api/simulations/start** -- start(Context ctx):
       - Parse JSON body: { deckName: string, gameCount: number, opponentDeckNames?: string[] }
       - Validate gameCount is one of [10, 50, 100, 500], return 400 if not
       - Load test deck via DeckSerializer (same pattern as WebServer.handleStartGame)
       - If opponentDeckNames provided, load those specific decks. Otherwise, load ALL decks in same format (user decks from deck directory + bundled AI decks from res/ai/decks/ -- reuse DeckHandler's deck listing logic)
       - Filter out the test deck from opponents
       - If no opponents found, return 400 "No opponent decks available"
       - Call SimulationRunner.startSimulation(deckName, testDeck, opponentDecks, gameCount)
       - Return 200 with { id: job.getId() }

       **GET /api/simulations/{id}/status** -- status(Context ctx):
       - Get job from SimulationRunner.getJob(id)
       - If null, check for saved result file (completed simulations)
       - Return 200 with SimulationSummary JSON

       **POST /api/simulations/{id}/cancel** -- cancel(Context ctx):
       - Call SimulationRunner.cancelJob(id)
       - Return 200

       **GET /api/simulations/history/{deckName}** -- history(Context ctx):
       - Read deck directory for files matching sim-{deckName}-*.json
       - Parse each file, return list sorted by timestamp descending
       - Return 200 with List of { id, timestamp, gamesCompleted, gamesTotal, winRate, eloRating }

       **DELETE /api/simulations/{id}** -- deleteResult(Context ctx):
       - Find and delete the sim-*.json file matching the id
       - Return 204

       **Result persistence:**
       - When SimulationJob completes (or is cancelled with partial results), save SimulationSummary as JSON
       - File path: same directory as deck files, named sim-{deckName}-{timestamp}.json
       - Use the deck directory path from ForgeConstants or the same path DeckHandler uses
       - The saving should happen in SimulationRunner when the job completes (add a completion callback)
