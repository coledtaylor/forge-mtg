    ```typescript
    export interface CardMatch { name: string; quantity: number }

    export interface DeckComposition {
      removal: { hard: CardMatch[]; soft: CardMatch[]; sweepers: CardMatch[]; total: number }
      ramp: { creatures: CardMatch[]; artifacts: CardMatch[]; spells: CardMatch[]; total: number }
      draw: { draw: CardMatch[]; cantrips: CardMatch[]; filtering: CardMatch[]; total: number }
    }

    export interface InteractionRange {
      creatures: number
      enchantments: number
      artifacts: number
      planeswalkers: number
      graveyards: number
      lands: number
    }

    export interface ConsistencyMetrics {
      fourOfRatio: number       // 0-1 (cards with 4+ copies / unique cards)
      tutorCount: number        // cards with "search your library"
      threatRedundancy: number  // distinct win condition cards
    }

    export interface WinConditionAnalysis {
      altWinCons: CardMatch[]
      bigThreats: CardMatch[]   // CMC 5+ creatures with power 4+
      planeswalkers: CardMatch[]
      total: number
    }
    ```

    **Functions to implement:**

    1. `analyzeDeckComposition(cards: DeckCardEntry[]): DeckComposition`
       - Use regex patterns from RESEARCH.md for removal (hard/soft/sweepers), ramp (creatures/artifacts/spells by typeLine), draw (draw/cantrips/filtering)
       - Guard: `if (!card.oracleText) continue` to skip basic lands
       - Cards CAN appear in multiple categories (Cryptic Command = removal AND draw). This is correct.
       - `total` on each category = sum of quantities across subcategories

    2. `analyzeInteractionRange(cards: DeckCardEntry[]): InteractionRange`
       - Count cards (by quantity) that match each permanent type answer pattern
       - Patterns: creature answers, enchantment answers (include "permanent"/"nonland permanent"), artifact answers (include "permanent"/"nonland permanent"), planeswalker answers, graveyard answers, land answers
       - A card answering multiple types counts in each (e.g., Vindicate = creature + enchantment + artifact + planeswalker + land)

    3. `analyzeConsistency(cards: DeckCardEntry[]): ConsistencyMetrics`
       - fourOfRatio: `cards.filter(c => c.quantity >= 4).length / cards.length` (0 if empty)
       - tutorCount: count cards (quantity) with `/search your library/i` in oracleText
       - threatRedundancy: reuse win condition count from analyzeWinConditions logic

    4. `analyzeWinConditions(cards: DeckCardEntry[]): WinConditionAnalysis`
       - altWinCons: cards with "you win the game" in oracle text
       - bigThreats: creature typeLine + cmc >= 5 + power >= 4
       - planeswalkers: "planeswalker" in typeLine
       - total = unique cards across all subcategories (deduplicate)

    **Regex patterns (from RESEARCH.md, use case-insensitive):**

    Hard removal: `/destroy target (?:.*?)(?:creature|permanent|planeswalker)/i`, `/exile target (?:.*?)(?:creature|permanent|planeswalker)/i`
    Sweepers: `/destroy all (?:.*?)creature/i`, `/exile all (?:.*?)creature/i`, `/deals? \d+ damage to each creature/i`, `/all creatures get -\d+\/-\d+/i`
    Soft removal: `/deals? \d+ damage to (?:target|any target)/i`, `/gets? -\d+\/-\d+/i`, `/target (?:.*?) fights? /i`, `/fight target/i`, `/return target (?:.*?) to its owner's hand/i`
    Mana producers: `/\{T\}:? add \{/i`
    Land search: `/search your library for (?:a|an|up to \w+)(?:\s\w+)* (?:basic )?land/i`
    Ritual: `/add \{[WUBRGC]\}\{[WUBRGC]\}/i`
    Draw multiple: `/draw (?:two|three|four|five|six|seven|\d+) cards?/i`
    Cantrip: `/draw a card/i`
    Filtering: `/\b(?:scry|surveil)\b/i`, `/look at the top \d+ cards?/i`

    Interaction range patterns from RESEARCH.md (creature, enchantment, artifact, planeswalker, graveyard, land answers).

    Exclude lands from ramp analysis (per user constraint: ramp = acceleration beyond land drops; subcategories are creatures, artifacts, spells only).
